Business Intent Title:
Add Priority Filter To Service Request Tracker
 
Business Idea:
The Service Request Tracker currently shows all service requests together. Platform engineers need a simple way to filter requests by priority so they can focus on high-priority work first.
 
Persona:
Platform Engineer
 
User Need:
As a platform engineer, I want to filter service requests by priority so that I can quickly focus on high-priority requests.
 
Current Context:
The starter app already has a service request list. Each request has a priority field such as low, medium, or high.
 
In Scope:
- Add a priority filter to the request list
- Allow filtering by all, low, medium, and high
- Show only matching requests after a filter is selected
- Keep the existing request creation flow unchanged
- Keep the existing backend API unchanged if frontend-only filtering is sufficient
 
Out Of Scope:
- Authentication
- Role-based access
- Advanced search
- Sorting
- Pagination
- Database changes
- New backend endpoint unless absolutely required
 
Constraints:
- Keep the change small enough for a single pull request
- Prefer frontend-only implementation if possible
- Do not introduce a new frontend framework
- Do not change the existing data model
- Add or update a simple test only if the repo already supports it
 
Expected Outcome:
Platform engineers can select a priority filter and see only requests matching that priority.
